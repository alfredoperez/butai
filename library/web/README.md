# @butai/library-web — GSAP recipe library + demo harness

The motion recipe set as **data**: one markdown-with-frontmatter file per recipe,
a generated catalog (JSON + MD), and a runnable vanilla-TS demo harness so every
flagship recipe can be seen, not just read.

## What lives here

- `recipes/<category>/<id>.md` — 57 recipes across `svg-diagram`, `text`,
  `transitions`, `attention`, `background`, `utility`. Frontmatter carries the
  `PatternMeta` fields (`id`, `name`, `kind: motion`, `category`, `engine`,
  `description`, `motion`, `tags`); the body holds the copy-ready snippet
  (first fenced code block) and an `## Adapt` note reworded for the platform:
  step/active-driven, reduced-motion safe.
- `recipes/INTENT-MAP.md` — the intent → recipe routing table (the decision layer),
  ported from the upstream motion catalog.
- `catalog/catalog.json` + `catalog/catalog.md` — generated by `pnpm gen`,
  committed, and byte-identical on regeneration (no timestamps).
- `demos/` — Vite + vanilla TS harness. `pnpm build` bundles it to `dist-demos/`;
  the index lists every cataloged recipe and mounts registered demos into a stage.
- `e2e/` — Playwright smoke (chromium): index lists ≥50 recipes, a demo run
  mutates the stage, zero console errors.

## Commands

```sh
pnpm --filter @butai/library-web gen    # regenerate catalog/ from recipes/
pnpm --filter @butai/library-web build  # tsc (src + tooling typecheck) + vite build
pnpm --filter @butai/library-web test   # vitest (catalog coverage, determinism)
pnpm --filter @butai/library-web e2e    # playwright smoke against vite preview
```

## Recipe ids and provenance

Ids preserve the upstream harvest ids (`svg-01-…`, `t-09-…`, `ui-13-…`, `adv-05-…`).
Source: the deck-native GSAP recipe reference in
the platform's internal presentations app (`apps/presentations/docs/gsap-recipes.md`, itself harvested from
an 84-entry GSAP prompt library), restructured here one-file-per-recipe. The
snippets are short GSAP API idioms. `svg-09-blob-morph-loop` is filed under
`background` (its actual role) rather than `svg-diagram`.

## Intentionally left behind

The upstream "Omitted" section is **not** ported — those effects have no clean
mapping to a stepped, non-scrolling surface:

- **Scroll-mechanism**: SCR-04/05 parallax layers, SCR-07 pinned section,
  SCR-08 pinned sequence with snap, SCR-09 horizontal scroll section,
  SCR-10 scroll velocity skew, SCR-12 scroll class toggle, SCR-14 scroll-snapped
  sections, SCR-15 scroll progress indicator, UI-12 navbar hide on scroll.
- **Web-app UI**: UI-15 skeleton loader (nothing here is async-loaded).
- **Cursor flourishes**: the whole CUR-01…09 family (followers, trails,
  spotlight, hover-expand) — a projected surface has no meaningful cursor. The
  `gsap.quickTo` + desktop-guard pattern survives in the pointer recipes
  (`ui-02`, `ui-08`, `adv-02`).

If one of these ever earns a place, start from the upstream doc above.
