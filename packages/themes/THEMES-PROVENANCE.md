# Theme provenance

Every theme shipped in this package is our own original work. This file is the
licensing gate for `packages/themes`: it records how originality was
determined, per-theme evidence for everything ported, what was deliberately
excluded, and which themes were authored fresh for butai.

**Source repo (read-only):** the platform's internal presentations app (`apps/presentations/src/themes/`)
(13 CSS files, 55 `[data-theme]` blocks total). Only 9 of those 55 blocks are
ours; the other 46 are bulk imports from a third-party repo and are FORBIDDEN
here.

## How provenance was determined

Three checks, applied to every upstream theme:

1. **File header comments.** The two aggregator files open with explicit port
   attributions:
   - `frontend-slides.css` (line 1–2): *"Themes ported from
     zarazhangrui/frontend-slides (12 style presets)."*
   - `frontend-slides-bold.css` (line 1–3): *""Bold pack" themes ported from
     zarazhangrui/frontend-slides (bold-template-pack). 34 templates →
     [data-theme] blocks."*

   Anything in those two files is EXCLUDED — with one exception:
   `midnight-coral` (block at ~line 893 of `frontend-slides-bold.css`), whose
   own comment reads *"example (create-theme skill)"* — it was generated by our
   own create-theme skill as the file's worked example, so it is original.
2. **Standalone file + no port attribution + author-voice comments.** The
   eight standalone theme files carry first-person design commentary (e.g.
   `brand.css`: *"The default theme. Editorial and premium, not corporate."*)
   and no attribution to any external source — original.
3. **Git history.** Verified in the source repo on 2026-07-07: the originals
   enter at the 2026-02-21 theme-system commit `4e5593e` ("feat: presentations
   app with theme system…") or in later first-party feature commits
   (`brand.css` via `ebff70c`, 2026-06-22; `dark.css` last touched by
   `8c3a237`, 2026-05-25). The entire third-party pack arrived in a single
   bulk commit `a87f92b` (2026-07-01, "feat(presentations): slide platform —
   kit, studios, motion system…") — both `frontend-slides.css` and
   `frontend-slides-bold.css` have exactly that one commit in their history.

## Gray zone — `primitives.css` (deferred, NOT ported)

`primitives.css` is our own reusable implementation, but of the bold pack's
signature DEVICES (hard-offset shadows, sticker borders, Win95 bevels). Per
the phase plan it is NOT ported in P1; the decision is logged in
`_ops/GAPS.md` for the Morning Queue. Likewise `base.css` was not ported (it
`@import`s the excluded files).

## New originals — authored fresh for butai (P1)

Four themes written from scratch for this package (inspired by OUR artifacts,
copied from nothing). Each carries the header comment
`/* butai original — authored for butai P1 */`, the full required token set,
the recommended font tokens, and at least one signature device rule:

| Theme | Mode | Derivation |
|---|---|---|
| `blueprint` | dark | Authored from our own internal `frame.md` design language: near-black `#0f0f13` ground, faint 44px hairline grid + vignette (signature device), glowing blue structure, one scarce yellow accent, JetBrains Mono chrome. |
| `atelier` | light | Light editorial sibling of our `brand` lineage: warm paper, ink text, single terracotta accent, serif display; signature device: terracotta brushstroke rule under headlines. |
| `stage` | dark | The butai identity theme (舞台, "stage"): deep charcoal, warm gold accent; signature device: fixed spotlight radial falling from top center. |
| `marker` | light | From our own diagram brand (cream ground, black 2px structure, teal/blue/gold/coral accents); signature device: asymmetric "hand-drawn wobble" border radii. Deliberately avoids the bold pack's hard-offset-shadow look. |

None of the new names collide with any EXCLUDE-list id.
